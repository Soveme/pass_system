# Техническое задание - Требования

## Основные требования ТЗ

### 1. Функциональные требования

#### 1.1 Получение пропуска гостем
- ✓ Гость получает короткую ссылку на мобильное устройство
- ✓ Ввод ФИО, компании, телефона, фото (опционально)
- ✓ Генерация QR-кода с датой и временем действия
- ✓ Отправка пропуска по email и SMS

#### 1.2 Проверка пропуска охраной
- ✓ Сканирование QR-кода мобильным устройством
- ✓ Проверка срока действия
- ✓ Отображение: имя, фото, статус (Разрешён/Запрещён)
- ✓ Автоматическая фиксация времени входа/выхода

#### 1.3 Администрирование
- ✓ Веб-панель с фильтрацией по дате, имени, компании
- ✓ Просмотр карточки посетителя
- ✓ Управление сроками и статусами пропусков

#### 1.4 Интеграция и уведомления
- ✓ Интеграция с Google Calendar и Outlook
- ✓ Push-уведомления о сроке истечения
- ✓ Тревожные сигналы при нарушениях

### 2. Требования к безопасности

- ✓ Соответствие ФЗ-152 и GDPR
- ✓ AES-256 шифрование ПДн
- ✓ HTTPS/TLS для передачи
- ✓ RBAC: охранник, HR, администратор, IT-специалист, руководство
- ✓ Логирование аудита
- ✓ Анонимизация данных старше 1 года

### 3. Производительность

- ✓ Среднее время выдачи пропуска: 60 секунд
- ✓ Вероятность ошибочного пропуска: ≤ 0,1%
- ✓ Поддержка 5000 одновременных пользователей
- ✓ Пиковая нагрузка: 100 сканирований в минуту
- ✓ Обновление статистики: не реже чем раз в минуту

### 4. Интеграции

- ✓ Google Calendar
- ✓ Microsoft Outlook
- ✓ Email (SMTP)
- ✓ SMS (Twilio - заглушка)

## Развертывание

- Docker & Docker Compose
- PostgreSQL 14+
- Redis 7+
- Python 3.11+
- Node.js 18+
